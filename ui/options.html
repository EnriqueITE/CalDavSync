Welcome to CalDavSync

by Enrique Serrano Aparicio

CalDavSync mirrors your local Thunderbird calendar to a private CalDAV server. Thunderbird may show a broad full-access permission because this add-on uses a Thunderbird Experiment API. Please read the following before using it.

How it works

  • Events are pushed from Thunderbird to your CalDAV server. Sync is one-directional — changes on the server are not pulled back.
  • Deleting an event in Thunderbird will delete it on the server on the next sync.
  • A configurable Delete guard blocks sync if more than a set percentage of your events would be deleted at once — protecting you from accidental bulk wipes. You can adjust the threshold in settings or bypass it entirely with Force sync.
  • Your credentials are stored securely in Thunderbird's native Password Manager.

About the Thunderbird permission

  • Thunderbird shows the full-access warning for add-ons that use Experiment APIs.
  • CalDavSync needs that API to read the selected local calendar and use Thunderbird's Password Manager.
  • It does not read mail, contacts, local files, telemetry, or analytics.
  • Calendar data and CalDAV credentials are sent only to the HTTPS CalDAV collection URL you configure.

              Thunderbird Experiment API
              
Required to read the selected local calendar and to store the CalDAV password in Thunderbird's native Password Manager.

            

            

              Network access
              
Used only for validation and sync requests to the HTTPS CalDAV collection URL configured here.

            

            

              Local storage
              
Stores settings and sync state. The CalDAV password is not stored in extension storage.
